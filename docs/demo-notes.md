# Demo Script: Alaska Coastal Fishery Explorer

Use this guide to walk a viewer through the primary value propositions of the application in under 3 minutes.

## 1. The Opening Hook (30 seconds)
> "Managing Alaska's fisheries is a massive geographic challenge involving dozens of agencies. This app, the Fishery Explorer, pulls together disparate data from state and federal sources into one high-performance map."

*   **Show**: The default PWS view.
*   **Explain**: Mention the Node.js proxy/cache that ensures the map stays fast even when government servers are slow.

## 2. Deep Drill-Down (45 seconds)
> "We can move from high-level regions down to specific management units."

*   **Action**: Select **Southeast Alaska** region.
*   **Action**: Switch the layer to **Sections** or **Statistical Areas**.
*   **Explain**: "Users can see exactly which specific geographic definitions are being used for active management."

## 3. The Human Context (45 seconds)
> "Fisheries aren't just lines; they are connected to people."

*   **Action**: Search for **"Cordova"**.
*   **Action**: Click the community point to load the Census profile and TIGER boundary.
*   **Explain**: "We're dynamically pulling socioeconomic data from the U.S. Census ACS API to give every management area a human context."

## 4. Multi-Agency Context (30 seconds)
> "Finally, we can layer on federal environmental data."

*   **Action**: Select **Upper Cook Inlet**.
*   **Action**: Toggle **NOAA Beluga Critical Habitat**.
*   **Action**: Toggle **USGS River Gages**.
*   **Explain**: "Now we can see state management, federal endangered species habitat, and real-time USGS river flow data all in one view. This is where the real cross-agency analysis happens."

## 5. Closing
> "The goal is a single, unified source of truth for the Alaska coastline."
